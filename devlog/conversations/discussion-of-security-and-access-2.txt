You can sign commits using GPG, SSH, or S/MIME. SSH is often the simplest, especially if you already use SSH keys for authentication.

I see. What is "forwarding the agent"?
Finished

SSH agent forwarding allows your local ssh-agent (which holds your decrypted private keys) to be used through an SSH connection. This means you can authenticate from a remote server (e.g., a jump box or cloud instance) to another server (e.g., GitHub) using your local keys — without storing private keys on the remote machine.
How It Works:

    You connect to Server A with ssh -A or ForwardAgent yes.

    From Server A, you SSH into Server B or access GitHub.

    The authentication request is securely forwarded back to your local ssh-agent.

    Your private key never leaves your machine. 

Enable It:

# One-time (command line)
ssh -A user@server

# Permanent (in ~/.ssh/config)
Host server
  ForwardAgent yes
 

    ⚠️ Only enable on trusted servers — anyone with root access can misuse your forwarded agent.

Ohhh.... I see. Yes, dynamic library loading comes with a heavy cybersecurity burden.
Finished

Yes, dynamic library loading (via dlopen()) is a major cybersecurity risk when untrusted or unvalidated libraries are loaded — exactly what happens in CVE-2023-38408.

The vulnerability exploits the fact that ssh-agent allows loading PKCS#11 modules from system paths like /usr/lib. Even though these are "system" libraries, they weren't designed to be loaded into a security-critical process like ssh-agent.

Attackers chain side effects from loading/unloading these libraries — such as executable stacks, signal handlers, or NODELETE flags — to:

    Bypass ASLR, PIE, and NX protections.

    Corrupt memory and redirect execution flow.

    Achieve remote code execution (RCE) on your machine. 

This is why modern mitigations:

    Disable remote PKCS#11 loading by default (OpenSSH 9.3p2+).

    Let you start ssh-agent with an empty allowlist: ssh-agent -P ''.

    Recommend avoiding agent forwarding on untrusted hosts. 

Dynamic loading is powerful — but trust boundaries must be enforced.
