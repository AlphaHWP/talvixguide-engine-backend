🔐 Cryptography & Trust Model
Signing & Verification

    Options:

        OpenSSL: RSA/ECDSA (familiar, widely supported)

        Libsodium: Ed25519 (modern, faster, safer defaults)

    Recommendation: Prefer Ed25519 via Libsodium for new systems unless PKI integration requires RSA.

Trust Model

    Explicit key trust (out-of-band verification)

    Future: Optional JWKS support for key discovery

Signature Scope

    Support both:

        File-wide signatures (entire metadata blob)

        Per-entry signatures (fine-grained trust)

// Pseudocode: Sign metadata
auto signature = crypto::sign(
    metadata_json, 
    private_key, 
    algorithm::ED25519
);

Yes — CRTP and heavy template metaprogramming can significantly increase compile times, especially with deep or repeated instantiations. However, the performance benefits at runtime (zero-overhead polymorphism) often justify the cost.

The extern template keyword (C++11) helps reduce compile time and binary size by preventing redundant template instantiations across translation units. You declare extern template class MyTemplate<int>; in headers and provide the instantiation (template class MyTemplate<int>;) in one .cpp file.

✅ Best Practice:

    extern template for commonly used types

    Precompiled headers

    Separate compilation of template-heavy components
