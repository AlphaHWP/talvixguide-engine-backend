Finished

For your signing key pair, the Ed25519 algorithm is the recommended choice.
Why Ed25519?

    Security: Ed25519 is based on the Edwards-curve Digital Signature Algorithm (EdDSA) and offers strong security with resistance to side-channel attacks.

    Performance: It is faster than RSA and ECDSA in both key generation and signing operations.

    Key Size: Keys are smaller (256 bits) yet more secure than equivalent RSA keys (e.g., 4096-bit RSA).

    Modern Standard: Widely supported in OpenSSH (6.5+) and recommended by security experts. 

How to Generate

ssh-keygen -t ed25519 -C "your_email@example.com"
 

    ⚠️ Note: While Ed25519 is preferred, ensure your environment supports it. Legacy systems (e.g., older Linux distributions or Microsoft Azure) may not support Ed25519. In such cases, use RSA 4096-bit as a fallback:

    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
     

SSH Agent Setup (for convenience)

To avoid entering your passphrase repeatedly:

eval "$(ssh-agent -s)"

# Add your key
ssh-add ~/.ssh/id_ed25519
 

On macOS, you can persist the key in your keychain:

ssh-add --apple-use-keychain ~/.ssh/id_ed25519

The core.sshCommand Git configuration overrides the default SSH command used for git fetch, git push, and other network operations.

When set, Git uses your specified command instead of the default ssh. This lets you:

    Use a custom private key via -i /path/to/key
