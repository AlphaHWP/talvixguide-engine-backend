What is package-lock.cmake?

package-lock.cmake is a dependency lock file used by CPM.cmake to ensure reproducible builds.

    It records the exact versions (e.g., Git tags, commits) of all dependencies.

    Generated via the cpm-update-package-lock CMake target.

    Should be committed to version control.

    Allows dependency overrides without changing CMakeLists.txt.

Example entry:

CPMDeclarePackage(
  Catch2
  VERSION 3.4.0
  GITHUB_REPOSITORY catchorg/Catch2
)
 

Use in CMakeLists.txt:

include(cmake/CPM.cmake)
CPMUsePackageLock(package-lock.cmake)
 

What is Renovate?

Renovate is an automated dependency update tool that:

    Scans your project for outdated dependencies.

    Creates pull requests to update them.

    Supports many ecosystems (npm, pip, Maven, etc.).

For C++/CPM:

    Renovate doesn’t natively support package-lock.cmake.

    Use regex managers in renovate.json to parse CPMDeclarePackage() entries.

    Can auto-update VERSION, GIT_TAG, or GITHUB_REPOSITORY fields.

Example config:

{
  "regexManagers": [
    {
      "fileMatch": ["(^|/)package-lock\\.cmake$"],
      "matchStrings": [
        "CPMDeclarePackage\\s*\\(\\s*\\w+\\s*.*VERSION\\s+\"?(?<currentValue>.*?)\"?\\s+GITHUB_REPOSITORY\\s+\"?(?<depName>.*?)\"?"
      ],
      "datasourceTemplate": "github-releases"
    }
  ]
}
 

This lets Renovate automatically update your C++ dependencies when new releases are available.

For human-edited metadata files, TOML is recommended over YAML.
Why TOML?

    Less error-prone: Not sensitive to indentation (unlike YAML, where whitespace errors break parsing).

    Simpler syntax: Explicit structure with clear section headers ([section]) and key-value pairs.

    Predictable: No ambiguous parsing rules or hidden gotchas (e.g., NO interpreted as false in YAML).

    Designed for config: TOML was created specifically for configuration files, while YAML is a general-purpose serialization format.

    Growing adoption: Used by Rust (Cargo), Python (Poetry), and Buildpacks — ideal for developer-facing metadata. 

When to Use YAML?

Only if you need deeply nested structures or multi-line strings, and can enforce strict linting.

For your use case — creators manually writing metadata — TOML is safer, clearer, and more beginner-friendly.
