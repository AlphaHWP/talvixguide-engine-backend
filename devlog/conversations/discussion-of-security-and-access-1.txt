Hmm... I tried to push and I got something like "authentication is not supported for git operations". What does this mean?
Finished

The error "authentication is not supported for git operations" occurs because GitHub no longer accepts account passwords for Git operations (since August 13, 2021). You must use either a Personal Access Token (PAT) or SSH key instead.
🔐 Solution 1: Use a Personal Access Token (PAT)

    Go to GitHub Settings → Developer Settings → Personal Access Tokens.

    Click Generate new token (classic).

    Set a note (e.g., "Git CLI"), expiration, and select the repo scope.

    Generate and copy the token (you won’t see it again).

    When Git prompts for a password, paste the token instead. 

    💡 To avoid entering credentials every time:

git config --global credential.helper store
 

Then perform one git push and enter your username + token. Git will save them.
🔑 Solution 2: Use SSH (Recommended)

    Generate an SSH key (if you don’t have one):

    ssh-keygen -t ed25519 -C "your_email@example.com"
     

    Start the SSH agent and add your key:

    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
     

    Add your public key to GitHub:

        Copy the public key: cat ~/.ssh/id_ed25519.pub

        Go to GitHub SSH and GPG keys, click New SSH key, paste it, and save. 

    Ensure your repo uses SSH URL:

    git remote set-url origin git@github.com:username/repo.git
     

Now git push will work without passwords or tokens.

What is ed25519?
Finished

Ed25519 is a modern elliptic curve digital signature algorithm based on the Curve25519. It offers:

    Strong security: Resistant to side-channel and timing attacks due to constant-time operations.

    High performance: Faster than RSA and ECDSA, with shorter keys (256-bit) and signatures.

    Compact keys: Public and private keys are small, improving efficiency.

    No weak parameters: Designed without arbitrary constants, reducing risk of backdoors. 

It's the default SSH key type in OpenSSH (since version 6.5) and widely recommended for secure, efficient authentication.

The SHA256 and randomart are visual and cryptographic tools to help you identify and verify SSH keys.
🔐 SHA256 Fingerprint

    A cryptographic hash of your public key.

    Uniquely represents the key — even a tiny change creates a completely different hash.

    Used to verify a key’s authenticity (e.g., when connecting to a server). 

🎨 Randomart Image

    A visual representation of the SHA256 fingerprint.

    Generated using the "Drunken Bishop" algorithm — a path drawn on a 9×17 grid based on the hash.

    Designed to make it easier for humans to spot differences between keys at a glance. 

    Example: Two similar fingerprints produce very different images, helping detect spoofing or errors. 

You can view them later with:
